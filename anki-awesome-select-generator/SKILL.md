# Anki题目导入文件生成器

## 核心流程

### 1. 收集用户需求（必须执行，不可跳过）

**⚠️ 重要：在执行任何转换之前，必须使用 question 工具向用户确认以下选项。这是强制步骤，不可跳过或假设默认值。**

使用 question 工具一次询问以下问题：

**a. 牌组类型**
- 新牌组：从ID 1开始
- 追加牌组：读取用户提供的已有import文件，获取最大ID并递增

**b. 章节处理方式（多章节时）**
- 独立输出：每个章节生成单独的txt文件
- 合并输出：所有章节合并到一个txt文件

**c. 标签策略（两步询问）**

**第一步：解析输入文件 + 同步生成建议一级标签**
- 读取题目内容，分析学科/主题
- 生成建议的一级标签（如"马克思主义哲学"）

**第二步：询问用户标签选择**（同时展示建议标签）
```
question: "根据题目内容，建议使用一级标签：'马克思主义哲学'。请选择标签策略："
options:
  - 使用建议标签
  - 自定义一级标签（将提示输入）
  - 不使用标签
```

**第三步：若选择使用标签（建议或自定义），询问是否生成子标签**
```
question: "是否在该一级标签下自动生成二级、三级标签？"
options:
  - 自动生成（根据题目内容分析知识点，生成结构化标签如：马克思主义哲学::唯物辩证法::矛盾论）
  - 不生成（仅使用一级标签）
```

**d. 答案解释（notes字段）**
- 先检测输入文件是否已包含答案解释字段
- 若已包含：直接使用
- 若未包含：询问用户是否补充答案解释
  - 是：搜索相关资料补充（⚠️ 会消耗较多token）
  - 否：留空

### 2. 解析输入文件

支持格式：
- **xlsx**：使用openpyxl读取，识别列：题号、类型、题目、选项、答案
- **toml**：使用tomllib读取，结构：questions[].{number, type, stem, answer, options}
- **csv**：标准CSV格式
- **txt**：自定义分隔符格式

### 3. 输出格式规范

Anki导入文件格式（TSV）：
```
id	question	options	answer	notes	tags
```

**字段说明：**
- `id`：唯一递增数字，全局不重复
- `question`：题干 + `{{c1::【】}}` 占位符
- `options`：选项内容用 `||` 分隔（不要带A./B./C./D.前缀，模板会自动显示选项标签）
- `answer`：答案序号（单选=数字如`1`，多选=数字如`1||3||4`，判断=`1`或`2`，数字对应选项位置）
- `notes`：答案解释（如题目解析、知识点说明等）
- `tags`：标签（章节名或AI生成的知识点标签）

**题目格式化规则：**
- 单选题：`题干内容{{c1::【】}}`
- 多选题：`题干内容{{c1::【】}}`
- 判断题：`题干内容{{c1::【】}}。`

### 4. 完成后提醒

输出文件后，提醒用户：
> 导入Anki时请选择 **AwesomeSelect-3.x** 模板

## 脚本使用

对于批量处理，可使用 `scripts/convert.py`：
```bash
python3 scripts/convert.py --input <文件路径> --output <输出路径> --start-id <起始ID> --tag <标签>
```

## 多文件合并

合并多个章节时：
1. 按章节顺序处理
2. ID全局递增，不重复
3. 每个文件无表头行
4. 直接追加到合并文件

## 多级标签生成（可选）

Anki支持多级标签，使用 `::` 作为分隔符。

当用户选择自动生成子标签时，分析题目内容生成：
- 一级标签：学科大类（如"马克思主义哲学"）
- 二级标签：知识模块（如"唯物辩证法"）
- 三级标签：具体知识点（如"矛盾的普遍性与特殊性"）

标签用 `::` 分隔，示例：`马克思主义哲学::唯物辩证法::矛盾论`

**标签规则：**
- 所有标签都在用户选择的一级标签下
- 二级、三级标签根据题目内容自动生成
- 若用户选择不生成子标签，则只使用一级标签
- 多个标签用空格分隔，每个标签都必须以一级标签开头（如：`马克思主义哲学::唯物论`）
- 标签层级内不得包含空格，章节名中的空格应替换为下划线（如：`第七章` 而非 `第七章 共产主义`）
- 生成标签时，章节标签和知识点标签应分开：章节标签为 `一级标签::章节名`，知识点标签为 `一级标签::知识点层级`
