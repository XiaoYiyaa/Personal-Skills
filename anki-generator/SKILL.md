# Anki题目导入文件生成器

## 核心流程

### 1. 收集用户需求（必须执行，不可跳过）

**⚠️ 重要：在执行任何转换之前，必须使用 question 工具向用户确认以下选项。这是强制步骤，不可跳过或假设默认值。**

使用 question 工具一次询问以下三个问题：

**a. 牌组类型**
- 新牌组：从ID 1开始
- 追加牌组：读取用户提供的已有import文件，获取最大ID并递增

**b. 章节处理方式（多章节时）**
- 独立输出：每个章节生成单独的txt文件
- 合并输出：所有章节合并到一个txt文件

**c. 标签策略**
- 默认：使用章节名作为标签（如"第一章"、"导论"）
- AI自动标签：根据题目内容分析知识点，生成结构化标签（如"马克思主义哲学/唯物辩证法/矛盾论"）

### 2. 解析输入文件

支持格式：
- **xlsx**：使用openpyxl读取，识别列：题号、类型、题目、选项、答案
- **toml**：使用tomllib读取，结构：questions[].{number, type, stem, answer, options}
- **csv**：标准CSV格式
- **txt**：自定义分隔符格式

### 3. 输出格式规范

Anki导入文件格式（TSV）：
```
id	question	options	answer	notes
```

**字段说明：**
- `id`：唯一递增数字，全局不重复
- `question`：题干 + `{{c1::【】}}` 占位符
- `options`：选项内容用 `||` 分隔（不要带A./B./C./D.前缀，模板会自动显示选项标签）
- `answer`：答案序号（单选=数字如`1`，多选=数字如`1||3||4`，判断=`1`或`2`，数字对应选项位置）
- `notes`：标签（章节名或AI生成的知识点标签）

**题目格式化规则：**
- 单选题：`题干内容{{c1::【】}}`
- 多选题：`题干内容{{c1::【】}}`
- 判断题：`题干内容{{c1::【】}}。`

### 4. 完成后提醒

输出文件后，提醒用户：
> 导入Anki时请选择 **AwesomeSelect-3.x** 模板

## 依赖说明

本 skill 生成的导入文件依赖 Anki 选择题模版：
- 仓库地址：[git9527/anki-awesome-select](https://github.com/git9527/anki-awesome-select)
- 模板名称：AwesomeSelect-3.x
- 安装方式：从仓库下载并导入 Anki

## 脚本使用

对于批量处理，可使用 `scripts/convert.py`：
```bash
python3 scripts/convert.py --input <文件路径> --output <输出路径> --start-id <起始ID> --tag <标签>
```

## 多文件合并

合并多个章节时：
1. 按章节顺序处理
2. ID全局递增，不重复
3. 每个文件无表头行
4. 直接追加到合并文件

## AI标签生成（可选）

当用户选择AI自动标签时，分析题目内容生成：
- 一级标签：学科大类（如"马克思主义哲学"）
- 二级标签：知识模块（如"唯物辩证法"）
- 三级标签：具体知识点（如"矛盾的普遍性与特殊性"）

标签用 `/` 分隔，示例：`马克思主义哲学/唯物辩证法/矛盾论`
