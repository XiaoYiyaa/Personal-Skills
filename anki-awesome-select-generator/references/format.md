# Anki导入格式规范

## 文件格式

TSV（Tab分隔），无表头行

## 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| id | 唯一递增数字 | 1, 2, 3... |
| question | 题干 + cloze占位符 | 马克思主义的基本立场是{{c1::【】}} |
| options | 选项内容（不带A./B.前缀），用\|\|分隔 | 意识第一性的\|\|物质是第一性的 |
| answer | 正确答案的**位置序号**（不是字母） | 单选: 1 / 多选: 1\|\|3\|\|4 / 判断: 1或2 |
| notes | 答案解释 | 意识是物质的反映... |
| tags | 标签（多级用`::`，多标签用空格分隔） | 马克思主义哲学::唯物论 |

## 题型格式

### 单选题
```
question: 题干内容{{c1::【】}}
options: 选项内容A||选项内容B||选项内容C||选项内容D
answer: 1 (对应第一个选项)
notes: 答案解释（如有）
tags: 标签
```

### 多选题
```
question: 题干内容{{c1::【】}}
options: 选项内容A||选项内容B||选项内容C||选项内容D
answer: 1||2||4 (对应第一、二、四个选项)
notes: 答案解释（如有）
tags: 标签
```

### 判断题
```
question: 题干内容{{c1::【】}}。
options: 对||错
answer: 1 (对) 或 2 (错)
notes: 答案解释（如有）
tags: 标签
```

## 答案解释（notes字段）

转换时需检查输入文件是否包含答案解释：
- **xlsx**：检查第6列（F列）是否有内容
- **toml**：检查questions[].notes字段是否存在
- **csv/txt**：检查是否有额外列包含解释内容

若输入文件无答案解释，需询问用户是否补充：
- 是：搜索相关资料补充（⚠️ 会消耗较多token）
- 否：留空

## 导入设置

1. 打开Anki → 文件 → 导入
2. 选择txt文件
3. 勾选 **允许HTML**
4. 分隔符选择 **Tab**
5. 模板选择 **AwesomeSelect-3.x**
6. 导入

## ID规则

- 全局唯一，不可重复
- 追加牌组时，从已有最大ID + 1 开始
- 建议使用递增数字

## 标签规则

Anki支持多级标签，使用 `::` 作为分隔符。

**标签策略：**
1. 分析输入内容生成建议的一级标签
2. 用户可选择：使用建议标签 / 自定义一级标签 / 不使用标签
3. 若使用标签，可选择是否自动生成二级、三级标签

**标签示例：**
- 单个标签：`马克思主义哲学`
- 多级标签：`马克思主义哲学::唯物辩证法::矛盾论`
- 多个标签：`马克思主义哲学::唯物论 马克思主义哲学::矛盾论`（空格分隔，每个标签以一级标签开头）
